# 第 12 章：结课展示与项目档案

## 最终提交物

完整项目应包含：

- 可运行网页；
- 资料卡片文件；
- 检索和回答逻辑说明；
- 10 个测试问题；
- 失败案例表；
- 课堂展示脚本；
- 项目反思。

## 项目评分表

| 维度 | 说明 |
| --- | --- |
| 资料质量 | 片段清楚，有来源和更新时间 |
| 检索效果 | 能解释候选资料排序原因 |
| 回答可信度 | 引用支持结论，不越界 |
| 拒答设计 | 资料不足或高风险时能拒答 |
| 失败案例改进 | 能根据失败样例修改系统 |
| 展示表达 | 能讲清楚系统流程和边界 |

## 作品集写法

如果把项目放到代码仓库，项目说明不应只写“用了检索增强生成”。更合适的结构是：

1. 项目解决的问题；
2. 系统流程图；
3. 数据样例；
4. 交互截图；
5. 测试问题与结果；
6. 已知问题和下一步计划。

## 最终提醒

可信人工智能问答项目的重点不是让模型看起来强，而是让回答可检查、可追溯、可改进。
