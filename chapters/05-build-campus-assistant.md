# 第 5 章：动手做校园资料问答助手

## 项目结构

本章对应 [互动实验](https://sherkevin.github.io/ai-evidence-quest/labs/playground.html)。它用浏览器里的 JavaScript 实现最小 RAG 流程。

核心数据结构是一组资料卡片：

```js
{
  id: "A1",
  title: "图书馆开放时间",
  text: "学校图书馆周一至周五 8:00-21:00 开放...",
  keys: ["图书馆", "开放", "周末"]
}
```

## 实现步骤

1. 准备资料卡片；
2. 为每张卡片写关键词；
3. 根据问题和关键词重合计算分数；
4. 选择分数最高的资料；
5. 用资料生成回答；
6. 分数太低时拒答。

## 分层挑战

### 基础

新增一条“食堂开放时间”资料，让系统能回答食堂问题。

### 进阶

把“借多久”和“借期”设为同义词，让检索更接近真实语言。

### 挑战

设计一个 badcase 表格，记录系统答错的问题、错误原因和改进方法。

| 问题 | 错误类型 | 改进方法 |
| --- | --- | --- |
| 我可以借东西吗？ | 问题过于模糊 | 引导学生补充对象 |
| 周日图书馆开吗？ | 需要识别周日 | 加入周日关键词 |

## 展示要求

展示时不要只演示“成功回答”，还要展示一次拒答和一次 badcase 改进。
