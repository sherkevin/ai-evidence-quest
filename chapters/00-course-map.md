# 第 0 章：课程地图与项目成品

## 本书的最终作品

本项目最终完成一个浏览器里的“校园资料问答助手”。它不是接入真实学校系统的产品，而是一个教学原型，用来呈现 AI 问答系统背后的结构。

成品至少包含四个能力：

1. 能接收一个校园规则问题；
2. 能从资料库中找出可能相关的片段；
3. 能根据资料生成带引用的回答；
4. 当资料不足时，不编造答案。

## 推荐课时

| 课时 | 内容 | 产出 |
| --- | --- | --- |
| 1 | 问题、资料与可信回答 | 判断回答是否有证据 |
| 2 | 搜索与相似度 | 解释为什么某段资料排在前面 |
| 3 | RAG 流水线 | 画出问答助手流程图 |
| 4 | 引用、拒答与风险 | 设计拒答规则 |
| 5 | 项目实现 | 完成可运行原型 |
| 6 | 展示与复盘 | 展示 badcase 和改进方案 |

## 学习方式

每章都按同一节奏组织：

- 先提出一个真实问题；
- 再给出最小概念；
- 接着操作一个小实验；
- 最后用任务单记录观察和改进。

这种结构参考了“动手学”类书式课程：概念不脱离实验，实验不脱离可解释的任务。
