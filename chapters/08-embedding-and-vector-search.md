# 第 8 章：Embedding 与向量检索

## 从关键词到语义

关键词搜索只能看词是否重合。问题“借多久”和资料“借期 30 天”没有完全相同的词，但语义相关。Embedding 的作用就是把文字变成向量，让意思相近的句子在空间里更近。

可以先用一个课堂比喻：

> 每句话都被放到一张地图上。意思接近的句子站得近，意思不同的句子站得远。

## 不急着调用大模型

本课可以分两层：

- 第一层：用手工特征模拟 embedding，例如是否涉及时间、地点、人物、规则；
- 第二层：用真实 embedding API 或本地模型替换手工特征。

这样安排的好处是先理解结构，再替换工具。

## 小实验

设计 4 个问题：

1. 图书馆周末开吗？
2. 周日能去图书馆吗？
3. 一次能借几本书？
4. 社团申请教室要提前多久？

先用关键词匹配排序，再讨论如果使用 embedding，哪些问题的效果可能提升。

## 反思题

- embedding 会不会把看起来相近但风险不同的问题放得太近？
- 为什么检索结果仍然需要人工检查？
