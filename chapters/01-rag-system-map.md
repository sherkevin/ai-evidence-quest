# 1. 检索增强生成系统地图

可信问答系统不是“把问题直接交给大模型”。一个可检查的回答系统至少包含五个环节：

```text
资料来源 -> 资料切分 -> 检索排序 -> 生成回答 -> 引用与评价
```

本章先建立系统地图。后续每个实验都会回到这张图：答案从哪里来，证据在哪里，系统为什么选择这些资料，什么时候应该拒答。

## 五个核心部件

| 部件 | 作用 | 课堂可观察现象 |
| --- | --- | --- |
| 资料库 | 保存校历、社团章程、活动说明、课程介绍 | 资料是否完整会直接影响回答 |
| 切分器 | 把长文档拆成可检索片段 | 切得过长会混入噪声，切得过短会丢上下文 |
| 检索器 | 找出与问题相关的片段 | 关键词检索和语义检索会给出不同结果 |
| 生成器 | 根据证据组织回答 | 没有证据时不能编造 |
| 评价器 | 记录引用、置信度、失败类型 | 系统改进需要数据，而不只是感觉 |

## 课堂任务

选择一份校园资料，例如图书馆规则或社团教室申请说明，完成以下表格。

| 问题 | 需要的资料 | 可能命中的关键词 | 是否可能需要拒答 |
| --- | --- | --- | --- |
| | | | |

完成后，把资料写成 3-5 张资料卡片。每张卡片需要包含：

- 编号：例如 A1、A2。
- 标题：一句话概括资料主题。
- 正文：只保留回答问题需要的信息。
- 来源：资料来自哪里。
- 更新时间：判断资料是否过期。

## 教师备课提示

课堂正文只保留适合高中项目课的概念：资料来源、资料切分、检索排序、引用依据、拒答边界和结果评价。教师备课时可以把复杂工程实现作为背景知识，但课堂展示应始终回到可观察、可操作、可解释的任务。
