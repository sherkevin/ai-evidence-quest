# 5. 课程项目路线

本课程围绕“校园资料问答助手”逐步完成。每一阶段都产生一个可以检查的项目产物，而不是只停留在概念讲解。

## 四个版本

| 版本 | 核心能力 | 项目产物 |
| --- | --- | --- |
| V0 资料卡片版 | 人工整理 10-20 张资料卡 | 资料表、问题清单、来源记录 |
| V1 关键词检索版 | 根据关键词召回资料 | 检索日志、命中和漏召案例 |
| V2 混合检索版 | 合并关键词、语义、来源可靠度和时间因素 | 排序解释表、失败案例库 |
| V3 可信回答版 | 生成回答并展示引用、拒答和置信度 | 可演示问答助手、评价报告 |

## 每一课的固定节奏

1. 观察一个真实问题：从校园场景提出可检索问题。
2. 处理一份资料：整理、切分、标注来源和更新时间。
3. 调整一个参数：关键词权重、语义权重、返回条数或拒答阈值。
4. 记录一次结果：命中资料、回答质量、失败类型和改进建议。

| | | | | |

失败类型建议使用以下标签：

- `资料缺失`：资料库里没有答案。
- `问题模糊`：问题缺少对象、时间或场景。
- `召回错误`：相关资料没有排到前面。
- `引用不足`：回答没有清楚标出来源。
- `越界问题`：问题超出课程资料范围。

## 结课作品要求

结课作品应包含四部分：可运行演示页、资料库、检索与回答日志、项目反思。评价重点是系统是否可解释、可改进，而不是界面是否炫技。
