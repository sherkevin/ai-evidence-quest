# 第 9 章：从关键词搜索到向量数据库

## 向量数据库解决什么问题

当资料只有 5 条，用数组遍历就够了。当资料有 5 万条，系统需要更快地找到相似向量。向量数据库负责保存向量、建立索引、快速返回 Top-K。

本章不要求部署复杂服务，而是让学习者知道向量数据库在 RAG 里的位置：

```text
资料 -> 切分 -> embedding -> 向量数据库 -> 检索 Top-K -> 回答
```

## 参考路径

Datawhale 的 `easy-vecdb` 把向量数据库拆成基础概念、向量嵌入、向量搜索、ANN 算法和工程应用。本项目借鉴这个顺序，但只保留高中课堂需要的最小链条：

- 为什么需要向量；
- 为什么需要索引；
- Top-K 是什么；
- Recall 和速度为什么有取舍。

## 动手任务

用表格模拟一个向量数据库：

| id | 主题 | 向量 | 文本 |
| --- | --- | --- | --- |
| A1 | 图书馆 | [1,0,1] | 周六开放，周日闭馆 |
| B1 | 社团 | [0,1,0] | 社团申请需提前 3 天 |

给定问题向量 `[1,0,0]`，计算它和每条资料的相似程度。

## 课堂讨论

- 如果资料很多，为什么不能每次从头查？
- 为什么向量数据库返回的是“候选”，不是最终答案？
- 如果 Top-K 里没有正确资料，后面的模型还能答对吗？
