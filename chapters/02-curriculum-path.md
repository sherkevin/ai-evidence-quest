# 2. 高中化改造路线

本书把上游的论文 RAG 系统改造成“校园资料问答助手”。这个主题足够真实，也足够可控：资料可以来自校历、社团章程、课程介绍、活动通知；问题来自校园生活；评价标准可以由课堂共同制定。

## 项目边界

本项目不追求一开始就接入最强的大模型，而追求三件事：

- 能把资料切成合适片段。
- 能检索到与问题相关的证据。
- 能根据证据回答，并在证据不足时拒答。

这三个目标比“回答看起来很聪明”更重要。它们对应真实 AI 应用中的可靠性、可解释性和安全边界。

## 四个渐进版本

| 版本 | 核心能力 | 可交付成果 |
| --- | --- | --- |
| V0 资料卡片版 | 人工整理 10-20 张资料卡 | 资料表、问题清单 |
| V1 关键词检索版 | 根据关键词召回资料 | 检索日志、命中分析 |
| V2 语义检索版 | 使用向量相似度找相近资料 | 相似度表、误召回案例 |
| V3 可信回答版 | 生成回答并展示引用、拒答 | 可演示的问答助手 |

## 每一课的固定节奏

1. 读一个真实工程片段：从上游项目中抽取对应模块。
2. 做一个小实验：在浏览器或轻量脚本中观察现象。
3. 改一处项目：把概念落到校园资料助手里。
4. 写一段项目日志：记录设计选择、失败案例和改进方案。

## 评价方式

结课作品不只看界面是否好看，而看四类证据：

| 评价项 | 观察点 |
| --- | --- |
| 资料建设 | 资料来源是否清楚，切分是否合理 |
| 检索质量 | 是否能解释为什么命中或漏掉某条资料 |
| 回答可信度 | 是否引用资料，是否避免编造 |
| 反思能力 | 是否能指出系统边界和改进方向 |

这样的评价方式更接近真实 AI 工程，也更适合展示“教学设计 + 项目实践”的能力。
