# 第 3 章：检索增强生成流水线

## 一句话解释

检索增强生成可以先理解成：

> 先查资料，再根据资料回答。

它通常包含四步：

1. 接收问题；
2. 检索资料；
3. 组织回答；
4. 检查回答是否有证据。

## 流程图

```text
用户问题 -> 检索资料 -> 选择证据 -> 生成回答 -> 引用来源 / 拒答
```

高中课堂不需要一开始讲向量表示细节，但要让学习者知道：生成式人工智能的回答应该受到资料约束。没有资料时，系统不应该假装知道。

## 动手任务

给定问题：

> 实验室可以一个人进入吗？

请写出一条回答，但必须满足：

- 至少引用一条资料；
- 不加入资料里没有的信息；
- 如果资料不足，要说明需要人工确认。

## 反思题

- 检索增强生成和普通聊天机器人最大的区别是什么？
- 如果检索到了错误资料，生成阶段还能救回来吗？
- 为什么最后一步要做证据检查？
